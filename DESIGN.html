Design system · v1 · 2026

Konak runs on warm parchment, a serif voice, and a single terracotta accent.

This document is the canonical reference for the Konak website (Belgrade). It mirrors the active Claude / Anthropic visual language and adapts it for a boutique-hospitality product. Anything you build here — landing pages, demos, partner pages — should pass through this guide first.

01 · Principles

Five rules that override everything else.

If a design choice contradicts one of these, the choice is wrong — not the rule.

Warm only

Every neutral has a yellow-brown undertone. No cool blue-grays. The page background is parchment (#f5f4ed), not white.

Serif for authority

All headlines are Iowan Old Style / Charter / Georgia at weight 500. Body is system sans. Mono for metadata, captions, code.

One accent, twice max

Terracotta (#c96442) belongs to eyebrows and the primary CTA. Anything else competes with it and loses.

Editorial pacing

Sections breathe like a magazine spread. Body line-height 1.6. Dark and light bands alternate to give the page chapters.

Honest content

No invented numbers, no stat-slop, no generic "Feature One." Belgrade specifics (Dorćol, Skadarlija, DOR-014) or a real placeholder — never filler.

Restraint over ornament

One decisive flourish per page. Landing gets the entryway illustration, how-it-works gets the dark quote band, demo gets the live phone walkthrough. That's the budget.

Posture

Most AI-product sites shout. Konak speaks at conversational volume — and trusts the reader to lean in.

The warmth of the palette, the weight of the serif, and the rhythm of the sections are the design — not the decoration on top of it.

05 · Components

The five primitives that build every page.

If a page needs a component that isn't here, add it to this document before using it twice.

Buttons

One primary per view. Secondary and ghost can repeat freely.

<a class="btn btn-primary">Try the demo</a>
<a class="btn btn-secondary">Read how it works</a>
<a class="btn btn-ghost btn-arrow">Learn more</a>

Cards

Three tiers: flat (no chrome), standard (Ivory + cream border), elevated (whisper shadow). Dark and tint variants exist for rhythm.

Standard card

Ivory background, 1px Border Cream, 16px radius. The default container for feature blocks and content groups.

Elevated

Adds a whisper shadow (0 4px 24px rgba(20,20,19,.05)). Reserve for one highlighted card per row.

Tint

Subtle terracotta wash. Use for a single brand-moment card per section — never a grid of them.

Dark

Used inside dark bands or as a quote/featured block. Body in Warm Silver.

Eyebrow + section header

The opening move of every section. Mono uppercase eyebrow, serif h2, optional lead paragraph.

03 · Partners

A curated set, walking-distance first.

Every Konak apartment carries its own list. Numbers vary by neighborhood and season.

Light / dark alternation

The page rhythm. A typical landing flows: hero (light) → features (light) → quote band (dark) → logos (light) → CTA (light). Never three same-theme sections in a row.

06 · Component reuse

Compose from what exists. Never rebuild primitives.

Scope: every change under src/ (SvelteKit). Static HTML prototypes (index.html, etc.) follow section 05. The app follows this section.

Instruction for LLMs

Before writing or editing any Svelte UI, run the mandatory workflow below. Import from the paths listed here. If nothing matches, add via shadcn-svelte into src/shared/components/ui/ — do not hand-roll table, form, dialog, or button primitives.

Mandatory workflow

  1. State the UI need (e.g. paginated table with row delete).
  2. Search existing code — list or grep src/shared/components/ui/, src/features/*/components/, and similar pages under src/routes/ or src/shared/components/pages/.
  3. Reuse the closest match — import it; extend via props and snippets; compose multiple existing pieces.
  4. Only if nothing fits: run bunx shadcn-svelte@latest add … (see components.json) or add a thin feature wrapper — never duplicate table/form/dialog logic.
  5. Copy a reference implementation when wiring Convex + UI (see decision table).

Canonical locations

src/shared/components/ui/ // shadcn-svelte + app composites (Button, DataTable, MutationForm, …) src/features/auth/components/ // login, sign-up, OTP, password reset, OAuth, logout src/features/uploadFile/components/ // single/multiple upload dropzones and file items components.json // shadcn aliases — ui → @/shared/components/ui

shadcn-svelte

Primitives live under @/shared/components/ui per components.json. Package manager: Bun — use bunx, not npx.

bunx shadcn-svelte@latest add button

import { Button } from '@/shared/components/ui/button/index.js';

import Link from '@/shared/components/ui/link/link.svelte';
<Link href="/en/dashboard">Dashboard</Link>

If button/, dialog/, or similar already exists, use it — do not add a parallel implementation. For in-app routes, always Link (locale-aware). Plain <a> only for external URLs (https://, mailto:).

Browse before inventing

List src/shared/components/ui/ when the table is not enough. Folders include: action-button, alert-dialog, alert-dialog-button, app-sidebar, avatar, breadcrumb, button, calendar, card, chart, checkbox, copy-button, data-table, date-picker, drawer, dropdown-menu, field, footer, header, input, label, mutation-form, paginated-data, pagination, popover, progress, section, sections, select, sheet, sidebar, skeleton, sonner, spinner, table, tabs, textarea, time-input, tooltip, and others.

Do / don't

Do

  • Import from @/shared/components/ui/… or feature component paths.
  • Extend via props, snippets, and composition.
  • Copy patterns from table/+page.svelte and admin user pages.
  • Add missing shadcn primitives with the CLI into ui/.
  • Use Link from link/link.svelte for in-app navigation (locale-aware).

Don't

  • Use a plain <a href="…"> for internal routes — use Link instead.
  • Hand-roll tables + delete buttons when DataTable + ActionButton exist.
  • Inline login, upload, or form markup in routes when feature components exist.
  • Install alternate UI kits or one-off dialog/form CSS in pages.
  • Add MyButton.svelte next to a route when Button already exists.

07 · Anti-patterns

Things that look right at first glance and aren't.

TrapWhy it's wrongDo instead
Bold serif headingsAll serifs stay at weight 500. Bold breaks the literary voice.Use size for hierarchy, not weight.
Drop shadows on cardsHeavy shadows feel SaaS-y. Konak uses ring shadows or none.box-shadow: 0 4px 24px rgba(20,20,19,.05) max.
Pure white page backgroundPure white kills the parchment personality.Always Parchment #f5f4ed.
Cool gray textEvery gray must have a yellow-brown tint.Olive Gray #5e5d59 or Stone Gray #87867f.
Inter as a display faceInter is body. Iowan/Charter/Georgia is display.Always --font-display for headlines.
Three accents on a screenBurns the accent budget.Eyebrow + one primary CTA. That's two. Stop.
Filler section to extend the pageEmpty sections feel padded.Cut the section. A shorter page beats a longer hollow one.
Hand-rolled Svelte UIDuplicates behavior and drifts from shadcn patterns.Section 06 — reuse DataTable, ActionButton, MutationForm, or add shadcn via CLI.

09 · For an LLM editing this site

Quick reference if you only read one section.

Editing src/ (Svelte)? Read 06 · Component reuse first — search existing components before writing UI.

Svelte app (src/)

Reuse @/shared/components/ui, auth and upload feature components. Tables → DataTable. Deletes → ActionButton. In-app links → Link (link/link.svelte), never raw <a>. Tokens → layout.css. See section 06.

Static HTML / marketing

Parchment background, Iowan/Charter serif headlines at weight 500, terracotta #c96442 as the only chromatic accent (eyebrow + one CTA per view), warm-toned grays only, 1.6 body line-height.

Never

Hand-rolled tables/forms/dialogs in Svelte; pure white background; bold serifs; cool blue-grays; generic SaaS gradients; emoji feature icons; invented stats; three accents on one screen.

If unsure

Svelte: open src/routes/(unprotected)/table/+page.svelte and copy its patterns. HTML: open index.html and copy its :root and class system.
